# 2.1 Movement, Interaction & Sensory Immersion

---

[](data:image/gif;base64,R0lGODlhAQABAIAAAP///wAAACH5BAEAAAAALAAAAAABAAEAAAICRAEAOw==)

Full VR locomotion via thumbsticks; optional teleportation for motion comfort.

Physical actions include climbing, swimming, crouching, crawling, jumping, and environmental traversal.

Fully tactile interaction: draw weapons from your sheath, swing a torch to clear webs, pour a drink into a cup, strike flint for fire.

Complex object manipulation includes: puzzle-solving, dismantling traps, opening chests, or using two-handed tools.

Environmental storytelling is accessed physically—read books, inspect worn letters, handle relics or totems.

### 🎮 Full VR Integration

- **Primary Interaction**: VR hand controllers and hand tracking.
- **Potential Enhancements**:
    - **Full-body tracking** for physical locomotion, martial arts, or dance-based magic.
    - Optional **voice recognition** for spells, dialogue, and commanding companions.
    - **Environmental Haptics** (if supported): touch, resistance, warmth.

### 🎛️ Control Scheme (Platform-Specific)

| Device | Controls |
| --- | --- |
| PC VR (e.g., Valve Index, Meta Quest via Link) | Thumbsticks for movement, triggers/grips for object use, gesture-based input for magic |
| Future Full-Body VR Platforms | Leg tracking for crouch, kick, balance; hip-based inventory; foot-based terrain interaction |

### 🗺️ Diagrams

Below are proposed diagrams and visual interfaces to support intuitive, immersive control and player interaction in VR. These will guide both UI/UX design and player onboarding.

---

### 🧭 1. **VR Control Map**

- **Locomotion Overlay** (hand/controller diagram):
    - Thumbstick: Walk / run (with analog pressure)
    - Button A: Interact / Confirm
    - Button B: Quick Menu / Cancel
    - Grip: Grab / Hold
    - Trigger: Use / Attack
    - Combined grip + gesture: Activate contextual tools (e.g., torch, journal)
- **Directional Gesture Grid** for gesture-based commands (e.g., point and swipe = give order)

---

### 🛡️ 2. **Combat & Interaction Zones (Full Body Tracking)**

- Diagram showing avatar zones:
    - **Head**: Helm, vision distortion, concussion feedback
    - **Chest**: Armor, breath exhaustion, spell impact
    - **Back**: Backpack access with over-shoulder grab
    - **Waist**: Quick inventory belt—grab sword, potion, or scroll
    - **Legs/Feet**: Used for kicks, dodging, kneeling, and crawling if full-body tracking is active

---

### 🧙 3. **Gesture-Based Spellcasting Wheel**

- Visual layout of radial spellcasting system using hand gestures:
    - Hold hand open → radial wheel appears with icons forming from light/smoke
    - Gesture direction or finger pose selects spell
    - Combine hand shape + motion = cast (e.g., circle motion with index finger = "shield")
- Optional: glyph tracing or spoken word to activate higher-tier spells

---

### 🎒 4. **Wearable & Diegetic Inventory**

- In-world inventory access:
    - Reach to belt → open pouch UI
    - Reach behind shoulder → access backpack (two-handed if large)
    - Holsters on thigh, back, or chest (weapons/tools)
- Diagram overlays show these access points on the VR player model

---

### 🧠 5. **NPC Memory & Relationship Visualization**

- Simple relationship web accessed via in-game “journal”:
    - Web of known NPCs showing **trust**, **fear**, **loyalty**, and **rumors**
    - Color-coded or mood-themed lines (e.g., red = betrayal risk, green = loyalty, gray = uncertain)
    - Hovering on NPC nodes reveals remembered actions/conversations

---

### 🕯️ 6. **Immersive Feedback Visual Cues**

- Diagram examples for:
    - **Health states**: blood drip overlay, breathing rate, screen desaturation
    - **Stamina/fatigue**: motion drag, head bob, limb tremble
    - **Magic overload**: sparks, aura distortion, voice feedback
