# 13. Accessibility & UX

### **🔰 Onboarding / Tutorials**

- **Progressive Introduction:**
    - The player begins in a guided safe space (e.g., a training grove, dream, or ancient temple), allowing time to learn controls without pressure.
    - Tutorials are contextual and layered: more advanced mechanics are introduced only after simpler ones are mastered.
- **Multi-modal Teaching:**
    - Visual cues (highlighted objects, glowing paths).
    - Audio instructions (spoken by NPCs or a guiding spirit).
    - Haptic feedback (in VR controllers) to indicate success/failure during tutorials.
    - Optional text captions or diagrams for hearing-impaired users.
- **Replayable Tutorials:**
    - All tutorials are available in the main menu and can be replayed individually.
    - Optional “Practice Zones” allow experimentation with skills (crafting, magic gestures, identification, combat, etc.) without consequences.
- **Tutorial Skipping & Resurfacing:**
    - Players can skip tutorials entirely and receive periodic reminders when struggling with a system.
    - Smart hints appear if the game detects repeated failed attempts (with user consent).

---

### **♿ Accessibility Features**

- **Motor Accessibility:**
    - One-handed play mode.
    - Adjustable gesture tolerance for magic, crafting, or combat input (e.g., simplified or alternate motions).
    - Auto-interact and magnetized object pickup for limited reach/motion.
    - Snap turning and teleportation movement options for motion sickness prevention.
- **Visual Accessibility:**
    - High-contrast mode and customizable UI color themes.
    - Colorblind support with presets (Protanopia, Deuteranopia, Tritanopia).
    - Text size, reticle size, and UI scaling adjustments.
    - Optional visual outlines for interactable objects.
- **Auditory Accessibility:**
    - Full closed captions with speaker identification and directional indicators.
    - Visual spellcasting indicators (e.g., glowing glyphs) for spell sounds.
    - Adjustable subtitle speed and font.
- **Cognitive & Neurodiversity:**
    - Optional simplified language mode.
    - Tutorial pacing controls (manual vs auto advance).
    - Highlighting key objects, characters, or tasks in cluttered environments.
    - "Task Focus Mode" reduces ambient distractions (noise, events).
- **Immersive Comfort:**
    - Customizable vignette for movement.
    - Motion blur toggle and camera shake reduction.
    - VR-safe camera transitions (no forced head movement).

---

### **🛠️ Custom Controls / Settings**

- **Control Mapping:**
    - Fully remappable inputs for motion controllers and traditional gamepads.
    - Profiles for common controller styles (e.g., Oculus Touch, Valve Index).
    - Presets for left-handed or hybrid input styles.
- **Interaction Modes:**
    - Choose between gesture, menu, or radial-wheel spellcasting.
    - Optional voice command system can be toggled or substituted with button input.
    - Alternative control schemes for crafting (realistic interaction vs. menu-based quick-craft).
- **User Interface Customization:**
    - UI transparency, element placement (floating or fixed in world), and size control.
    - Toggle immersion vs information: minimalist mode vs full HUD.
    - Optional floating guides (tooltips, objectives, NPC markers).
- **Save & Session Options:**
    - Auto-save frequency settings.
    - Manual save slots and “rest at safe zones” mechanic.
    - Session length pacing suggestions to promote VR safety.

- Haptic feedback
- Visual comfort (e.g., motion blur, blink teleportation)
- Colorblind modes, font scaling
- Locomotion alternatives (walk-in-place, seated)
