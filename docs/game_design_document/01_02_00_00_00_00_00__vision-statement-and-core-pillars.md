# 1.2 Vision Statement & Core Pillars

This section defines the stable design promise that later systems should serve. It should stay concise enough to read before a build session, but concrete enough to reject features that do not support the intended player experience.

The core vision is a VR-first systemic world where player action, environment, memory, survival pressure, social response, and legacy compound over time. The pillars below should be treated as design filters:

- Presence before abstraction: major interactions should be felt through embodied motion, diegetic tools, sound, haptics, and readable world feedback before they become menu operations.
- Living systems before scripted set pieces: weather, ecology, factions, memory, economy, and magic should expose reusable simulation rules that can create different outcomes under different conditions.
- Consequence before convenience: death, reputation, resource use, environmental damage, and promises made to NPCs should matter, while still giving players enough clarity to make informed decisions.
- Learning before leveling: player knowledge, observation, practice, and experimentation should carry real value instead of relying only on abstract stat growth.
- Small proof before grand scale: every pillar should first appear in a miniature harness or vertical slice before it becomes a planet-scale claim.

Implementation mapping: this section feeds the game repo vision brief, player experience contract, living-world simulation contract, and first-loop gameplay TODOs. New ideas added here should map to one of those surfaces or remain explicitly deferred.
